Boas Práticas

Regras do Cadastros Gerais para criação e formatação de scripts Oracle no schema INFOSAUDE.

Formatação e elaboração

1

Todo script deve ser criado/alterado usando o SQL Developer.

2

Não é permitido incluir comentários dentro do script (exceto COMMENT ON).

3

Utilizar / para separar comandos, sem espaços ou linhas em branco antes/depois.

4

O arquivo deve terminar com / na última linha.

5

Schema INFOSAUDE sempre informado nas instruções SQL.

6

Ponto e vírgula (;) apenas dentro de blocos PL/SQL.

7

Nomes de objetos não podem ultrapassar 30 caracteres.

Nomenclatura e objetos

8

CREATE SEQUENCE deve usar o prefixo SEQ_.

9

Constraints: PK_, FK_ e CKC_ com nomenclatura correta.

10

CREATE TABLE deve incluir COMMENT, GRANT e PUBLIC SYNONYM.

11

COMMENT ON TABLE deve conter a referência ao schema INFOSAUDE.

12

Arquivo gerado no padrão: CARD_<numero>_<tipo>_<campo>.sql.
